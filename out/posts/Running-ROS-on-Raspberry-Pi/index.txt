<p>To run ROS on a Raspberry Pi, we need to install Ubuntu because ROS isn&#39;t officially supported on Raspberry Pi OS. Given the hardware limitations of the Raspberry Pi 3 with only 1GB of RAM, it&#39;s not powerful enough to handle Ubuntu Core efficiently. Instead, we need to install <strong>Ubuntu Server</strong>, which is a lightweight alternative that can better accommodate the limited resources of the Raspberry Pi.</p>
<h2>Installing Ubuntu Server on Raspberry Pi</h2>
<h3>1. Download the Raspberry Pi Imager</h3>
<p>First, go to the <a href="https://www.raspberrypi.com/software/">official website</a> and download the Raspberry Pi Imager.<br><img src="/my-next-blog/images/1.png" alt=""></p>
<h3>2. Install the Imager</h3>
<p>After downloading the imager, open the file to start the installation process. Click <strong>Install</strong> and then <strong>Next</strong> when prompted.<br><img src="/my-next-blog/images/2.png" alt=""></p>
<h3>3. Launch the App</h3>
<p>Once the imager is installed, open the application.<br><img src="/my-next-blog/images/Pasted%20image%2020241202212428.png" alt="">    </p>
<h3>4. Select Your Device</h3>
<p>Choose the device you want to use. In my case, it&#39;s the <strong>Raspberry Pi 3</strong>.</p>
<h3>5. Choose the Operating System</h3>
<p>When selecting the OS, choose <strong>Other general-purpose OS</strong>.<br><img src="/my-next-blog/images/Pasted%20image%2020241202212609.png" alt=""><br>Then, select <strong>Ubuntu</strong>.    <img src="/my-next-blog/images/Pasted%20image%2020241202212735.png" alt=""><br>Ensure that you select <strong>Ubuntu Server</strong>.<br><img src="/my-next-blog/images/Pasted%20image%2020241202212911.png" alt=""><br>Make sure to choose the latest version of Ubuntu Server.    </p>
<h3>6. Select Storage</h3>
<p>Choose the storage device you want to use. Keep in mind that all data on the storage will be erased during the installation process, so select the correct device.<br><img src="/my-next-blog/images/Pasted%20image%2020241202213057.png" alt="">    </p>
<h3>7. Finalize and Install</h3>
<p>Once youâ€™ve selected the device and OS, click <strong>Next</strong>. A confirmation screen will appear.<br><img src="/my-next-blog/images/Pasted%20image%2020241203011921.png" alt="">	<strong>Note</strong>: Click Edit Settings <em>do not click No</em> you will be fucked down the line	<br><img src="/my-next-blog/images/Pasted%20image%2020241203012226.png" alt=""><br>In the <strong>GENERAL</strong> tab, enter your desired username and password. Changing the wireless LAN settings here won&#39;t affect the settings in the OS, so you can skip that part if needed.	In the</p>
<p><strong>SERVICES</strong> tab, ensure that <strong>Enable SSH</strong> is checked, and select <strong>Password Authentication</strong>.	 <img src="/my-next-blog/images/Pasted%20image%2020241203012505.png" alt=""><br>Once done, click <strong>Save</strong>, confirm with <strong>Yes</strong>, and wait for the download to complete.</p>
<p>Now you have Ubuntu Server installed on your SD card!</p>
<h3>8. Booting the Raspberry Pi</h3>
<p>Take the SD card that you&#39;ve prepared with Ubuntu Server and insert it into the Raspberry Pi. Connect the Raspberry Pi to a monitor, keyboard, and power source, then power it up. The system will perform some initial setup tasks, which you can ignore. After a short time, the Raspberry Pi will prompt you to log in. </p>
<ul>
<li>Use the same username and password you entered earlier to login.<br>You should now be logged into Ubuntu Server on your Raspberry Pi and ready to proceed with setting up ROS or other tasks.</li>
</ul>
<h2>Connecting Ubuntu Server to Wi-Fi</h2>
<p>Connecting Ubuntu Server to Wi-Fi without a GUI can be done using the terminal and a tool like <code>netplan</code>, which is included by default in recent Ubuntu versions. Hereâ€™s the easiest way to get connected:</p>
<h3>1. Identify Your Wireless Interface:</h3>
<ul>
<li>Run the following command to list your network interfaces:</li>
</ul>
<pre><code class="language-bash">ip link
</code></pre>
<ul>
<li>Look for an interface name like <code>wlan0</code> or <code>wlp3s0</code> (this is your wireless interface).</li>
</ul>
<h3>2. Edit the Netplan Configuration:</h3>
<ul>
<li>Open the Netplan configuration file for your system. The file is usually located <code>/etc/netplan/</code>. Use the following command to find it:</li>
</ul>
<pre><code class="language-bash">ls /etc/netplan/
</code></pre>
<ul>
<li>Open the configuration file (replace <code>filename.yaml</code> with the actual file name):</li>
</ul>
<pre><code class="language-bash">sudo nano /etc/netplan/filename.yaml
</code></pre>
<h3>3. Add Wi-Fi Configuration:</h3>
<ul>
<li>Update the file to include your Wi-Fi settings. Here&#39;s an example configuration:</li>
</ul>
<pre><code class="language-yaml">network:
  version: 2
  wifis:
    wlan0:  # Replace with your interface name
      dhcp4: true
      access-points:
        &quot;YourWiFiSSID&quot;:
          password: YourWiFiPassword
</code></pre>
<ul>
<li>make sure you copy the spaces and tabs correctly</li>
</ul>
<h3>4. Apply the Configuration:</h3>
<ul>
<li>Save and close the file (<code>Ctrl+O</code>, <code>Enter</code>, and <code>Ctrl+X</code> in <code>nano</code>).</li>
<li>Apply the new configuration:</li>
</ul>
<pre><code class="language-bash">sudo netplan apply
</code></pre>
<h3>5. Test the Connection:</h3>
<ul>
<li>Check if you are connected to Wi-Fi:</li>
</ul>
<pre><code class="language-bash">ip a
</code></pre>
<ul>
<li>Look for an IP address under your wireless interface.</li>
</ul>
